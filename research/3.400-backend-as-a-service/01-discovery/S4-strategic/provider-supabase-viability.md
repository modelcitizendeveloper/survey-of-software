## Lock-In Assessment

**Lock-In Score: 75/100** (MODERATE-HIGH)

### Lock-In Factors

1. **PostgreSQL (Low Lock-In):** Standard database, easy to migrate (8-16 hours to self-hosted PostgreSQL)
2. **Supabase SDKs (Moderate Lock-In):** Client libraries used throughout app (50-100 hours to rewrite)
3. **Row-Level Security (Moderate Lock-In):** RLS policies must be rewritten as API authorization (20-40 hours)
4. **Real-Time Subscriptions (Moderate Lock-In):** Real-time APIs are Supabase-specific (40-80 hours to replace with Pusher, Ably)
5. **Edge Functions (Low Lock-In):** Deno runtime, standard JavaScript (easy to migrate to Vercel, Netlify)

**Total Migration Time:** 80-120 hours ($8K-12K developer time at $100/hour)

### Migration Paths

**Supabase Cloud → Self-Hosted Supabase:**
- Time: 8-16 hours (Docker Compose setup, database export/import)
- Difficulty: LOW-MODERATE (same stack, just self-hosted)
- Cost: $50-200/month (VPS + S3 storage)

**Supabase → Standard PostgreSQL + PaaS:**
- Time: 80-120 hours (rewrite SDKs, RLS → API auth, real-time → Pusher)
- Difficulty: MODERATE (PostgreSQL same, but APIs need rewriting)
- Cost: $50-500/month (PaaS + PostgreSQL + S3)

**Mitigation:** Accept 75/100 lock-in. Migrate to self-hosted Supabase or standard PostgreSQL when costs exceed $500-1K/month.
