## 1. Company Overview

**Founded**: 2019
**Founders**: Karri Saarinen (CEO, ex-Airbnb), Tuomas Artman (CTO, ex-Uber)
**Status**: Private
**Valuation**: $400M (2023)
**Headquarters**: San Francisco, CA
**Employees**: ~50

### Market Positioning

"The issue tracking tool you'll enjoy using."

Linear positions itself as the **developer-focused project management** tool built by engineers, for engineers. Fast, keyboard-driven, minimal friction. The anti-Jira.

### Core Value Proposition

- **Speed**: Faster than any competitor (sub-100ms interactions)
- **Focus**: Keyboard-first, no distractions
- **Developer UX**: Built for how engineers actually work
- **Modern design**: Beautiful, clean, minimalist interface

## 2. Core Features

### Issue Tracking (Not "Tasks")
- **Issues**: Linear calls them "issues" not "tasks" (engineering terminology)
- **Labels**: Organize issues by type, feature area
- **Status**: Custom workflow states (Backlog, Todo, In Progress, Done)
- **Priority**: 4 levels (urgent, high, medium, low) with clear icons
- **Single assignee only**: ‚ùå Cannot assign to multiple people
- **Estimates**: Points or time estimates

### Multiple Views (5 View Types)
- **List view**: Default issue list (blazing fast)
- **Board view**: Kanban columns by status
- **Timeline view**: Gantt-style roadmap (basic)
- **Roadmap view**: High-level project planning
- **Active cycles view**: See current sprint work

**Note**: Fewer views than competitors (5 vs 15+ for ClickUp)

### Developer-Focused Features
- **Keyboard shortcuts**: Everything is keyboard-driven (Cmd+K)
- **GitHub integration**: Auto-link commits, PRs, branches
- **GitLab, Bitbucket**: Similar code integration
- **Figma integration**: Link designs to issues
- **Slack integration**: Create issues from messages
- **API-first**: Everything accessible via API

### Sprint Management (Cycles)
- **Cycles**: Linear's term for sprints (1-4 weeks)
- **Automatic cycle creation**: Rolling cycles
- **Velocity tracking**: See team throughput
- **Scope creep protection**: Warn when adding mid-cycle

### Collaboration Features
- **Comments & mentions**
- **Reactions**: Quick emoji responses
- **Threads**: Discussions on issues
- **Activity feed**: Real-time updates
- **Notifications**: Smart (not overwhelming)

## 4. Pricing Structure (2025)

### Free Plan
- **Cost**: $0
- **User limit**: Unlimited
- **Issue limit**: 250 issues (unlimited archived)
- **Features**:
  - 2 teams max
  - All integrations (GitHub, Slack, Figma)
  - API & webhooks
  - Roadmaps
  - Cycles (sprints)
  - Import/export
- **Limitations**:
  - Only 250 active issues
  - Only 2 teams
  - No advanced features

### Standard Plan
- **Cost**: $8/user/month (annual) or $10/user/month (monthly)
- **Features**:
  - Everything in Free, plus:
  - 5 teams
  - Unlimited issues
  - Triage (issue inbox)
  - Custom views
  - Advanced search
- **Best for**: Small engineering teams (5-20 people)

### Plus Plan
- **Cost**: $14/user/month (annual) or $17/user/month (monthly)
- **Features**:
  - Everything in Standard, plus:
  - Unlimited teams
  - Advanced Slack automations
  - Slack triage (create issues from reactions)
  - Detailed analytics
  - Issue templates
  - Project templates
- **Best for**: Growing engineering orgs (20-100 people)

### Enterprise Plan
- **Cost**: Custom pricing (contact sales)
- **Features**:
  - Everything in Plus, plus:
  - SAML/SSO
  - SCIM user provisioning
  - Advanced security controls
  - Dedicated support
  - SLA guarantees
  - Onboarding & training
- **Best for**: Large enterprises (100+ engineers) with security/compliance needs

### Cost Examples

| Team Size | Plan | Monthly Cost (Annual) | Annual Cost |
|-----------|------|----------------------|-------------|
| 5 users | Standard | $40 | $480 |
| 10 users | Standard | $80 | $960 |
| 20 users | Plus | $280 | $3,360 |
| 50 users | Plus | $700 | $8,400 |

**Note**: Linear is 27% cheaper than Asana Standard ($8 vs $11/user), similar to ClickUp

## 6. AI Capabilities (2025)

### Current AI Features

**Limited AI capabilities** (as of 2025):
- **Smart search**: Natural language issue search
- **Auto-complete**: Suggests labels, assignees based on patterns
- **Related issues**: AI suggests similar issues

**No AI features for**:
- Task generation
- Descriptions / summaries
- Prioritization
- Forecasting

### Why Limited AI?

Linear prioritizes **speed and focus** over AI features. Philosophy: "Ship software, don't fiddle with AI."

**Possible future** (speculation):
- AI-powered issue triage
- Automatic sprint planning
- Cycle scope prediction
- Issue dependency detection

### Comparison to Competitors

Asana: AI Studio, summaries, document analysis
ClickUp: ClickUp Brain ($7/user)
Linear: Minimal AI (intentional)

Linear users generally don't mind - they value speed > AI novelty.

## 8. Ideal Customer Profile

### Team Size Sweet Spot

**Best fit**: 5-50 engineers
- Dedicated engineering team (not cross-functional)
- Small enough for fast collaboration
- Large enough to benefit from structure

**Can work**: 3-100 engineers
- 3-5 engineers: Free plan sufficient (250 issues)
- 50-100 engineers: Multiple teams, roadmaps
- 100+ engineers: Enterprise plan

**Not ideal**: 1-2 engineers or 200+ engineers
- 1-2 engineers: Overkill (GitHub Issues sufficient)
- 200+ engineers: May need Jira for enterprise complexity

### Industry / Use Case Fit

**Best for**:
1. **Software engineering teams**: Backend, frontend, mobile, DevOps
2. **Product engineering**: Product + eng tightly coupled
3. **Startups**: Modern tools, fast-moving teams
4. **SaaS companies**: Continuous shipping, sprints/cycles
5. **Developer tools companies**: Dogfood-friendly

**Not ideal for**:
1. **Cross-functional teams**: Marketing, ops, sales can't use Linear
2. **Non-technical teams**: UI optimized for developers
3. **Complex portfolio management**: Need Jira/Asana for program-level
4. **Agencies**: Client work doesn't fit Linear's model

### Technical vs Non-Technical Users

**Technical users (developers)**: ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê (5/5)
- LOVE the keyboard shortcuts
- Speed is game-changing
- GitHub integration seamless
- Training time: <1 hour (intuitive for devs)

**Non-technical users**: ‚≠ê‚≠ê‚òÜ‚òÜ‚òÜ (2/5)

### Key Decision Criteria

Choose Linear if you value:
1. **Speed & developer UX** (fastest tool on market)
2. **Engineering-only team** (not cross-functional)
3. **Code integration** (GitHub/GitLab critical)
4. **Modern design** (aesthetics matter)
5. **Focus** (no feature bloat, no distractions)

Skip Linear if you need:
1. **Cross-functional teams** (marketing, ops, product all use same tool)
2. **Advanced portfolio management** (complex dependencies, resource planning)
3. **Multiple assignees** (Linear doesn't support it)
4. **Comprehensive reporting** (analytics weak)
5. **Wide integration ecosystem** (only 20 integrations)
