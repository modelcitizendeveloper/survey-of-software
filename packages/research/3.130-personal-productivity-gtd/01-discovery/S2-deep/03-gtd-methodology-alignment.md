# GTD Methodology Alignment Analysis

**How Well Does Each Tool Support Getting Things Done?**

**Reference:** David Allen's "Getting Things Done" (GTD) methodology

---

## GTD Methodology Overview

**The Five Steps:**
1. **Capture:** Collect what has your attention
2. **Clarify:** Process what it means
3. **Organize:** Put it where it belongs
4. **Reflect:** Review frequently
5. **Engage:** Simply do

**Key GTD Concepts:**
- Inbox (capture everything)
- Next Actions (very next physical action)
- Projects (outcomes requiring >1 action)
- Contexts (@home, @computer, @phone, @errands)
- Waiting For (delegated items)
- Someday/Maybe (future possibilities)
- Weekly Review (critical habit)
- Calendar (hard commitments only)
- Reference (support material)

## 1. Todoist - GTD Alignment

### Official GTD Status
‚úÖ **Officially Endorsed** - David Allen Company has created a setup guide for Todoist

### GTD Strengths

**Excellent Capture:**
- Quick add everywhere (mobile, desktop, browser extension)
- Natural language: "send invoice Friday at 3pm"
- Email forwarding to inbox
- Voice capture via Siri/Alexa/Google Assistant
- Browser clipper
- **Rating: 9/10**

**Good Clarify:**
- Easy to assign project, priority, labels, due date
- Comments for details
- Natural language reduces friction
- **Rating: 8/10**

**Strong Organize:**
- Projects (up to 300 on Pro)
- Labels = Contexts (@home, @work, etc.)
- Filters = Custom views
- Can create separate projects for:
  - Next Actions
  - Waiting For
  - Someday/Maybe
- **Rating: 8/10**

**Weak Reflect:**
- ‚ùå No dedicated review mode
- ‚ùå No review intervals
- Productivity trends show completion stats but not GTD-focused
- Must manually review projects
- **Rating: 5/10**

**Excellent Engage:**
- Today view
- Filters for custom Next Actions views
- Priority levels (P1-P4)
- Natural language makes adding new actions fast
- **Rating: 9/10**

### GTD Weaknesses

1. **No Review Mode:** Biggest gap - GTD's Weekly Review is critical
2. **No Defer Dates:** Can't hide tasks until ready (must use due dates or filters)
3. **Free Tier Insufficient:** Only 5 projects makes GTD impossible
4. **Requires Setup:** Need to create filters, label system, project structure

### GTD Workarounds

**Weekly Review:**
- Create recurring task "Weekly Review"
- Manually review each project
- Use filters to surface stale items

**Defer Dates:**
- Use due dates conservatively
- Create filter for "No date" to find Next Actions
- Use labels like @next_week to defer

**Someday/Maybe:**
- Create "Someday/Maybe" project
- Or use label @someday

### Overall GTD Rating: 8/10

**Excellent for:**
- Capture (best in class with natural language)
- Organization (labels + filters flexible)
- Engagement (Today view + filters)

**Missing:**
- Review mode (must do manually)
- Defer dates (can workaround)

**Best For:** GTD practitioners who want cross-platform, fast capture, and don't mind manual weekly review

## 2. Things 3 - GTD Alignment

### Official GTD Status
‚úì **Community Endorsed** - Not officially GTD-certified but widely used by GTD practitioners

### GTD Strengths

**Excellent Capture:**
- Quick Entry (keyboard shortcut everywhere)
- Inbox is prominent
- Natural language for dates
- Share sheet on iOS
- **Rating: 9/10**

**Excellent Clarify:**
- Easy to assign to project, area, tags, dates
- **Unique: "When" vs "Deadline"** - aligns perfectly with GTD's distinction between "next action date" and "hard deadline"
- Notes for details
- **Rating: 9/10**

**Excellent Organize:**
- **Inbox:** Dedicated inbox view
- **Projects:** Unlimited projects
- **Areas:** GTD's "Areas of Responsibility"
- **Tags:** Perfect for contexts (@home, @work, etc.)
- **Anytime:** Next Actions list
- **Upcoming:** Timeline view (Calendar + defer dates)
- **Someday:** Dedicated Someday/Maybe list
- **Headings:** Organize within projects
- **Rating: 9/10**

**Weak Reflect:**
- ‚ùå No dedicated review mode
- ‚ùå No review intervals
- ‚ùå No "last reviewed" dates
- Can use Logbook to see completed items
- **Rating: 5/10**

**Excellent Engage:**
- **Today:** Clear focus on today's actions
- **Anytime:** All available Next Actions
- Tags for context-based work
- Beautiful, distraction-free interface
- **Rating: 9/10**

### GTD Unique Advantage

**When vs Deadline (Do Date vs Due Date):**

Things 3 is the ONLY tool that explicitly separates:
- **When:** When you plan to START working on it (GTD: next action timing)
- **Deadline:** When it must be DONE (GTD: hard deadline)

This matches GTD philosophy perfectly - very few things have true deadlines, but many have planned start dates.

### GTD Weaknesses

1. **No Review Mode:** Like Todoist, missing dedicated GTD review
2. **Apple Only:** Can't use on Windows/Android
3. **No Natural Language:** Less sophisticated than Todoist for dates

### GTD Workarounds

**Weekly Review:**
- Create recurring task "Weekly Review" in Today
- Review each Area and Project manually
- Use Upcoming view to see timeline

**Contexts:**
- Use Tags for @home, @work, @computer, @errands, @phone
- Quick filter by tag

### Overall GTD Rating: 9/10

**Excellent for:**
- Organization (best GTD structure of any tool)
- When/Deadline distinction (unique)
- Clean, focused UX
- Apple ecosystem integration

**Missing:**
- Review mode (must do manually)
- Cross-platform (Apple only)

**Best For:** Apple users who want the cleanest, most GTD-aligned structure without complexity

## 3. OmniFocus - GTD Alignment

### Official GTD Status
‚úÖ **Purpose-Built for GTD** - Omni Group publishes "OmniFocus, GTD, and You" guide

### GTD Strengths

**Excellent Capture:**
- Inbox with quick entry
- Email to inbox
- Siri integration
- Quick Entry window (Mac)
- **Rating: 9/10**

**Excellent Clarify:**
- Comprehensive processing options
- Defer dates, due dates
- Flags for priorities
- Estimated duration
- Tags for contexts
- **Rating: 9/10**

**Excellent Organize:**
- Projects with sequential/parallel modes
- Folders = Areas of Responsibility
- Tags = Contexts (multiple per task)
- On Hold status for inactive projects
- **Perspectives** = Saved custom views for any GTD scenario
  - Can create: Next Actions, Waiting For, Someday/Maybe, Errands, etc.
- **Rating: 10/10**

**EXCELLENT Reflect:**
- ‚úÖ‚úÖ **Dedicated Review Mode** (unique to OmniFocus)
- Set custom review intervals per project/area
- Track last reviewed date
- Review Mode prompts you through each project
- **This is the KILLER FEATURE for GTD**
- **Rating: 10/10**

**Excellent Engage:**
- Forecast view (today + upcoming + calendar)
- Custom perspectives for any Next Actions view
- Flag for high-priority
- Multiple contexts via tags
- **Rating: 9/10**

### GTD Unique Advantages

1. **Review Mode:** ONLY tool with dedicated GTD Weekly Review feature
   - Set review frequency (weekly, monthly, etc.) per project
   - Shows "due for review" projects
   - Guides you through review process

2. **Custom Perspectives:** Create unlimited saved views
   - "Next Actions - Work" (available, tagged @work)
   - "Waiting For" (deferred items)
   - "Someday/Maybe" (on hold)
   - Any combination of criteria

3. **Forecast View:** See tasks + calendar events together (GTD: calendar for hard landscape)

### GTD Weaknesses

1. **Complexity:** Steep learning curve - overwhelming for beginners
2. **Apple-Centric:** Limited Windows/Android support
3. **Expensive:** $100/year vs $48 Todoist
4. **Overkill for Casual Users:** Too powerful if not fully committed to GTD

### GTD Best Practices in OmniFocus

**Projects:**
- Sequential for step-by-step (project planning, etc.)
- Parallel for independent actions (house maintenance, etc.)

**Perspectives:**
- Next Actions: Available, not on hold, not completed
- Waiting For: Deferred items with @waiting tag
- Errands: Available, tagged @errands
- High Energy: Available, tagged @high-energy

**Review:**
- Set most projects to weekly review
- Areas of Responsibility to monthly review
- Use Review Mode every Friday

### Overall GTD Rating: 10/10

**Excellent for:**
- Complete GTD implementation (all five steps)
- **Review Mode** (unique, critical)
- Custom Perspectives (infinite flexibility)
- Power users committed to GTD

**Tradeoffs:**
- Complexity (not beginner-friendly)
- Cost ($100/year)
- Apple ecosystem

**Best For:** Serious GTD practitioners who need maximum power and have mastered the methodology

## 5. Obsidian - GTD Alignment

### Official GTD Status
üîå **Plugin-Based** - Community GTD implementations via plugins

### GTD Strengths

**Moderate Capture:**
- Can capture tasks anywhere in notes
- Templates for quick capture
- Not as streamlined as dedicated task apps
- **Rating: 6/10**

**Good Clarify:**
- Tasks plugin supports dates, tags, recurrence
- Markdown for notes/details
- YAML frontmatter for metadata
- **Rating: 7/10**

**Excellent Organize:**
- Tasks queryable across entire vault
- Tags for contexts
- Notes for projects
- Dataview for powerful queries
- Can build sophisticated GTD structure
- **Rating: 9/10** (with plugins)

**Moderate Reflect:**
- Can create review note templates
- Dataview queries for stale items
- No automated prompts
- Requires discipline
- **Rating: 6/10**

**Good Engage:**
- Dataview queries for Next Actions
- Today note with task queries
- Tag-based context filtering
- **Rating: 7/10**

### GTD Plugin Stack

**Essential Plugins:**
1. **Tasks:** Enhanced task management
   - Due dates, start dates, recurrence
   - Query tasks across vault
   - Filters by status, tags, dates

2. **Dataview:** Query language for tasks
   - `LIST WHERE contains(tags, "@work") AND !completed`
   - Powerful but requires learning

3. **Calendar:** Daily notes integration

4. **Templater:** Advanced templates for capture

**GTD Vault Structure:**
```
/Inbox.md (capture)
/Projects/
  Project-A.md
  Project-B.md
/Areas/
  Work.md
  Personal.md
/Archive/
/Templates/
  daily-note.md
  weekly-review.md
```

### GTD Weaknesses

1. **Plugin Complexity:** Must understand Tasks, Dataview syntax
2. **Not Optimized for Tasks:** Obsidian is for notes first, tasks second
3. **Setup Required:** Significant time to build GTD system
4. **Maintenance:** Plugins update, may break workflows
5. **Learning Curve:** Markdown, YAML, Dataview query language

### GTD Workarounds

**Quick Capture:**
- Hotkey to append to Inbox.md
- Template for quick task entry
- Daily note for ephemeral capture

**Contexts:**
- Use tags: #context/work, #context/home, #context/errands
- Dataview query: `WHERE contains(tags, "context/work")`

**Review:**
- Weekly review template note
- Dataview query for projects not reviewed in 7 days

### Overall GTD Rating: 7.5/10

**Excellent for:**
- Data ownership (markdown files)
- Tasks + knowledge management unified
- Customization (build exactly what you want)
- Privacy (local-first)

**Weaknesses:**
- Steep learning curve
- Not optimized for quick task capture
- Requires technical comfort

**Best For:** Technical users who want tasks integrated with extensive note-taking/knowledge management and value data ownership

## GTD Methodology Gaps

### Missing from ALL Platforms

1. **Automatic Context Detection**
   - GPS-based: Show @errands when near shopping district
   - Calendar-based: Show @work during work hours
   - Device-based: Show @computer when at laptop

2. **Energy-Aware Scheduling**
   - Track when you have high/low energy
   - Suggest high-energy tasks during peak times

3. **GTD Coach/Assistant**
   - Prompt for Weekly Review if overdue
   - Suggest Next Actions based on context/time/energy
   - Detect stuck projects (no progress in X days)

4. **Collaborative GTD**
   - Share "Waiting For" items with teammates
   - Delegation tracking

### Missing from MOST Platforms

- **Review Mode:** Only OmniFocus has this
- **Defer/Start Dates:** Things 3, OmniFocus, Notion, Obsidian support; Todoist does not
- **When vs Due:** Only Things 3 explicitly separates
- **Natural Language:** Only Todoist has sophisticated NLP
