## Build vs. Buy Economics

### When Does Self-Hosted / DIY Make Sense?

**Framework:**
Compare 3-year TCO of SaaS vs. building with Tier 1 libraries (1.127 Financial Simulation)

---

### DIY Cash Flow Analysis: Tier 1 Libraries (1.127)

**Build Approach:**
- **Backend:** Python + pandas (data manipulation)
- **Financial calculations:** numpy-financial (NPV, IRR, PMT functions)
- **Forecasting:** statsmodels or Prophet (time series)
- **Database:** PostgreSQL (historical data)
- **Frontend:** React or Django templates
- **Charting:** Plotly or Chart.js

**Build Effort:**
- Initial development: 100-200 hours ($15K-30K at $150/hr)
- Integrations (QuickBooks API): 20-40 hours ($3K-6K)
- Ongoing maintenance: 20 hours/year ($3K/year)

**3-Year TCO (DIY):**
- Initial: $18K-36K
- Maintenance: $9K (3 years √ó $3K/yr)
- **Total: $27K-45K**

---

### SaaS 3-Year TCO Comparison

**Pulse:** $1,044 - $3,204 (Basics to Premium)
**Finmark:** $3,600 - $10,800
**Jirav:** $1,800 - $27,000
**Causal:** $18,000 - $36,000 (estimated)
**Mosaic:** $54,000+ (estimated)

---

### Build vs. Buy Breakeven Analysis

**DIY breaks even when:**
```
SaaS 3-year cost > DIY total cost ($27K-45K)

Breakeven monthly SaaS cost:
$27K / 36 months = $750/month
$45K / 36 months = $1,250/month
```

**Conclusion:**
- **Below $750/mo SaaS:** Buy (Pulse, Finmark, Jirav, Causal)
- **Above $1,250/mo SaaS:** Consider build (Mosaic, high-end enterprise)
- **Between $750-1,250/mo:** Depends on customization needs

---

### When to Build (DIY)

**Build makes sense if:**
1. **SaaS cost >$1,000/month** (3-year TCO = $36K+)
2. **Unique business model** (SaaS platforms don't fit your use case)
3. **You have engineering resources** (don't need to hire contractors)
4. **You value strategic control** (no vendor dependency)
5. **You're technical company** (can maintain indefinitely)

**Examples where build wins:**
- **FinTech startup:** Need custom financial models (regulatory, unique products)
- **Real estate developer:** Project finance is too specialized for generic tools
- **Private equity firm:** Portfolio-level consolidation with unique metrics
- **Internal tool for tech company:** Engineering bandwidth available, want full control

**Examples where buy wins:**
- **Small business:** No engineering resources, generic needs
- **Startup:** Speed > customization, engineering focused on product
- **Non-tech company:** Can't maintain custom software long-term
- **Accountant serving clients:** Need white-label, multi-client (build doesn't scale)

---

### Hybrid Approach: Excel/Sheets + Automation

**Middle Ground:**
- Keep financial model in Excel/Google Sheets
- Build lightweight automation:
  - Script to pull actuals from QuickBooks API (10-20 hours)
  - Script to update Excel with actuals (10-20 hours)
  - Scheduled task to run daily/weekly (5 hours)
- **Total effort:** 25-45 hours ($3,750-6,750)

**3-Year TCO (Hybrid):**
- Initial: $3,750-6,750
- Maintenance: $1,500/year (10 hours script updates)
- **Total: $8,250-11,250**

**Hybrid breaks even when:**
```
$8,250 / 36 months = $230/month
$11,250 / 36 months = $312/month
```

**Conclusion:**
- **Below $230/mo SaaS:** Just buy the SaaS (Pulse at $29-89/mo wins)
- **$230-750/mo range:** Hybrid might work (automation scripts + Excel)
- **Above $750/mo:** Consider full build

### Platform-Specific Lock-In Analysis

**Pulse (Very Low Lock-In):**
- CSV export available (standard format)
- Simple models (cash flow only, easy to rebuild)
- 1 integration (QuickBooks - easy to replicate)
- Escape cost: 5-10 hours ($750-1,500)
- **Mitigation:** Export monthly CSVs as backup

**Fathom (Low Lock-In):**
- Export available (likely CSV/Excel)
- 3-statement models (medium complexity)
- 6+ integrations (some rebuild needed)
- Escape cost: 10-20 hours ($1,500-3,000)
- **Mitigation:** Document custom KPIs, export templates

**Finmark (Low-Medium Lock-In):**
- Data export (assumed CSV)
- Hiring plans (medium complexity)
- 5 integrations (moderate rebuild)
- Escape cost: 10-20 hours ($1,500-3,000)
- **Mitigation:** Revenue-based pricing limits downside (low cost at low revenue)

**Jirav (Medium Lock-In):**
- Data export (assumed)
- AI-generated budgets (can't easily replicate)
- Multi-data source integrations
- White-label usage (if accountant channel, affects clients)
- Escape cost: 15-30 hours ($2,250-4,500)
- **Mitigation:** Export AI budgets as Excel baseline, rebuild incrementally

**Causal (Medium-High Lock-In):**
- Model export available
- Multi-dimensional models (complex, hard to rebuild)
- 15+ integrations (significant rebuild)
- Natural language formulas (unique to Causal)
- Version control (model history valuable)
- Escape cost: 20-40 hours ($3,000-6,000)
- **Mitigation:** Document model logic in plaintext, export monthly snapshots

**Mosaic (High Lock-In):**
- Data export (likely available)
- 150+ pre-built metrics (months to rebuild)
- 25+ integrations (extensive rebuild)
- Arc AI insights (can't replicate)
- Real-time dashboards (significant workflow dependency)
- Escape cost: 30-60 hours ($4,500-9,000)
- **Mitigation:** Document metric definitions, export data warehouse schema

**Dryrun (Medium-High Lock-In):**
- Export available (integration data + manual)
- Complex scenario models (hard to rebuild)
- Multi-entity consolidation (complex)
- Sage Intacct/Dynamics integration (mid-market ERP = sticky)
- Escape cost: 25-50 hours ($3,750-7,500)
- **Mitigation:** Maintain parallel Excel models (CFOs often do this anyway)

**PlanGuru (Low-Medium Lock-In):**
- Excel export (native format)
- Traditional forecasting methods (can rebuild in Excel)
- 2 integrations (QB, Xero - standard)
- Escape cost: 10-20 hours ($1,500-3,000)
- **Mitigation:** Excel export is escape hatch (low friction)

---

### Lock-In Mitigation Strategies

**Strategy 1: Regular Data Exports**
- Export monthly snapshots (CSV, Excel)
- Store in separate backup location
- Test import into alternative platform (annual test)
- **Cost:** 1 hour/month ($150/month)
- **Benefit:** Can switch platforms in days, not weeks

**Strategy 2: Document Custom Logic**
- Write plaintext descriptions of models, scenarios, metrics
- Screenshot key dashboards
- Export formula definitions
- **Cost:** 5-10 hours initial + 2 hours/quarter ($1,250 initial, $300/quarter)
- **Benefit:** Can rebuild platform-specific features

**Strategy 3: Maintain Parallel Excel Model (Hybrid)**
- Keep simplified Excel model updated monthly
- Use platform for day-to-day, Excel as backup
- **Cost:** 2-4 hours/month ($300-600/month)
- **Benefit:** Zero switching cost (Excel is backup system)

**Strategy 4: Choose Low Lock-In Platform**
- Accept fewer features in exchange for portability
- Pulse/Fathom/Finmark over Causal/Mosaic
- **Cost:** $0 (upfront choice)
- **Benefit:** Escape cost $750-3K vs. $4,500-9K

---

### Lock-In Decision Framework

**When to accept high lock-in (Causal, Mosaic):**
- Platform provides unique value (data warehouse integration, AI insights)
- Escape cost justified by time savings (20 hours/month saved = $3K/month value)
- Company has stable needs (not changing business model frequently)
- Vendor stability high (acquisition by Oracle/SAP = continuity)

**When to avoid high lock-in (choose Pulse, Fathom, Finmark):**
- Business model uncertainty (may pivot, need flexibility)
- Budget constraints (can't afford 20-40 hour migration later)
- Vendor stability concerns (newer company, market consolidation risk)
- Prefer optionality over features

## Future Trends (3-5 Year Outlook)

### Trend 1: AI Becomes Table-Stakes

**Current State (2025):**
- Only Mosaic (Arc AI) and Jirav (AI budgeting) have AI features
- Most platforms rely on traditional modeling

**3-5 Year Prediction:**
- All platforms add AI capabilities
- AI features:
  - Automated anomaly detection ("Revenue dropped 15% in product X")
  - Predictive forecasting (ML-based predictions, not just trends)
  - Natural language queries ("What's our cash runway?")
  - Auto-generated insights (proactive alerts)

**Strategic Implication:**
- Don't pay premium for AI today (will commoditize)
- Mosaic's Arc AI advantage shrinks as competitors add AI
- Wait 2-3 years, AI will be standard

---

### Trend 2: Real-Time Data Becomes Expected

**Current State (2025):**
- Most platforms batch-update daily or weekly
- Mosaic emphasizes "real-time" as differentiator

**3-5 Year Prediction:**
- Real-time data becomes standard (not differentiator)
- Driven by: API improvements, data warehouse maturity, customer demand
- "Monthly close" paradigm fades for tech companies

**Strategic Implication:**
- If you need real-time today, Mosaic is only option
- If you can wait 2-3 years, lower-tier platforms will catch up

---

### Trend 3: Data Warehouse Integration Expands

**Current State (2025):**
- Only Causal and Mosaic support Snowflake/BigQuery

**3-5 Year Prediction:**
- More platforms add data warehouse support
- Driven by: modern data stack adoption, customer data complexity

**Strategic Implication:**
- If you have Snowflake today, Causal or Mosaic only options

---

### Trend 4: Embedded Finance / FP&A

**Current State (2025):**

**3-5 Year Prediction:**
- QuickBooks adds cash flow forecasting (acquires or builds)

**Strategic Implication:**
- Standalone platforms must differentiate (advanced features, AI, integrations)

---

### Trend 5: Price Compression in Tier 1-2

**Current State (2025):**
- Pulse ($29-89/mo), Finmark (~$100-300/mo), Jirav ($50-750/mo)

**3-5 Year Prediction:**
