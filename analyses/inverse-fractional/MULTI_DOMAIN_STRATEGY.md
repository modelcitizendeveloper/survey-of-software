## Domain Expansion Decision Framework

### When to Launch New Domain

**Prerequisites** (all must be true):
1. **Previous domain validated** - 10+ bookings/month sustained for 2 months
2. **Audience research complete** - 5+ interviews with target role
3. **Content angles written** - Audience-specific copy for 3+ categories
4. **Case studies ready** - 1-2 examples with audience-specific angle
5. **Event pipeline** - 2+ events scheduled where audience attends

**Launch Criteria**:
- Domain setup (DNS, SSL, analytics): 2 hours
- Content deployment (audience-config.json): 4 hours
- QR postcard design: 3 hours
- Event testing: 1-2 events, 2-4 weeks
- Validation threshold: 5+ bookings in first month

**Decision Point**: If validation threshold NOT met, pause domain expansion and optimize existing domains

## Risk Mitigation

### Challenge: Domain Dilution

**Risk**: Too many domains dilutes brand, confuses SEO, splits traffic

**Mitigation**:
1. Launch domains sequentially (validate before expanding)
2. Canonical tags point to primary (CFO) domain for duplicate content
3. Cross-link between domains ("Also for CTOs: cto.inversefractional.com")
4. Monitor traffic split (ensure each domain drives minimum viable traffic)

---

### Challenge: Content Maintenance

**Risk**: Updating 4+ domains with every content change is time-consuming

**Mitigation**:
1. **Single source of truth** - All content in shared JSON files
2. **Domain detection** - Audience-specific copy loaded dynamically
3. **Component reuse** - Same components across all domains
4. **Automated deployment** - Single deploy updates all domains

---

### Challenge: Audience Mismatch

**Risk**: CTO visits CFO domain, doesn't find content relevant

**Mitigation**:
1. **Domain switcher** in header - "Switch to: CTO | CMO | CFO"
2. **Cross-links** in content - "Technical perspective: cto.inversefractional.com/fpa-platforms"
3. **QR codes target correctly** - CFO events → CFO domain, CTO events → CTO domain
4. **Google Search** should surface correct domain per query intent

## SEO Strategy

### Canonical URLs

**Problem**: Same content on multiple domains = duplicate content penalty

**Solution**: Use canonical tags to designate primary version

**CFO domain as primary** (most traffic initially):
```html
<!-- cfo.inversefractional.com/fpa-platforms -->
<link rel="canonical" href="https://cfo.inversefractional.com/fpa-platforms">

<!-- cto.inversefractional.com/fpa-platforms -->
<link rel="canonical" href="https://cfo.inversefractional.com/fpa-platforms">

<!-- cmo.inversefractional.com/fpa-platforms -->
<link rel="canonical" href="https://cfo.inversefractional.com/fpa-platforms">
```

**OR: Self-canonical** (if audiences are distinct enough):
```html
<!-- Each domain canonicalizes to itself -->
<link rel="canonical" href="https://{current-domain}/fpa-platforms">
```

**Decision**: Start with CFO as canonical, switch to self-canonical when CTO/CMO traffic > 20% of CFO
